# [IoT 제어 실습] 프로젝트 제안서

## 1. 과제명
**IoT 및 하이브리드 통신 기반 학생회관 강당 스마트 예약 및 실시간 상태 표시 시스템**
*(Smart Auditorium Reservation and Real-time Status Display System)*

---

## 2. 제안 배경 및 필요성
현재 학생회관 강당은 학생들의 동아리 활동, 학생회 회의, 학술 행사 등 다양한 목적으로 예약제 운영이 되고 있습니다. 그러나 다음과 같은 고질적인 문제점으로 인해 이용 학생들의 불편함이 지속되고 있습니다.

1. **실시간 사용 상태 확인 불가**: 강당이 예약제임에도 불구하고, 현재 예약 시간표나 사용 중 여부를 강당 외부(문 앞)에서 직관적으로 확인할 수 있는 현황판이 존재하지 않습니다.
2. **행사/회의 방해 우려**: 강당 안의 소리가 밖으로 잘 들리지 않는 경우, 학생들이 사용 중인지 확인하기 위해 직접 문을 열어 내부를 들여다보아야 하므로 안에서 진행 중인 행사나 회의 흐름을 방해하는 일이 번번이 발생합니다.
3. **온오프라인 정보 불일치**: 온라인 상에서의 예약 정보와 오프라인 현장 강당의 실시간 사용 상태가 유기적으로 연동되지 않아 예약 일정 관리에 비효율성이 따릅니다.

따라서 본 프로젝트는 **두 대의 라즈베리 파이(Raspberry Pi)**와 **센서 하드웨어 및 웹 기술**을 융합하여, 온라인 예약 편의성을 높이고 강당 외부에서 문을 열지 않고도 사용 상태를 직관적으로 판별할 수 있는 스마트 예약 제어 시스템을 제안합니다.

---

## 3. 연구 및 개발 목표
본 시스템의 핵심 목표는 웹 기반 예약 플랫폼과 오프라인 하드웨어 디스플레이 기기 간의 연동을 통해 강당 이용의 편의성을 극대화하는 것입니다.

* **온라인 예약 및 모니터링 뷰 제공**: 웹페이지를 통해 누구나 손쉽게 강당의 당일/주간 예약 현황을 조회 및 신규 등록할 수 있도록 합니다.
* **현장 직관성 강화 (LCD, LED, Buzzer 활용)**:
  - **LCD1602**: 강당 외부 벽면에 설치되어 현재 진행 중인 행사/회의명(예: `STATUS: IN MEETING`, `IoT Project Review`)을 실시간으로 표시합니다.
  - **RGB LED**: 강당의 상태를 색상으로 즉시 안내하여 먼 거리에서도 상태를 인지하게 만듭니다. (초록색: AVAILABLE / 파란색: RESERVED / 빨간색: IN_MEETING)
  - **Passive Buzzer**: 회의/행사의 정시 시작 및 종료 시간에 맞추어 멜로디(도-미-솔 / 솔-미-도)를 재생함으로써 강당 내외의 사용자들에게 시간 경과를 알립니다.
* **이중 장비 구축 및 분산 처리**: Apache 웹 서버와 하드웨어 제어/소켓 서버의 역할을 독립된 2대의 라즈베리 파이에 분담하여 성능 저하 및 데이터 병목을 예방합니다.

---

## 4. 연구 내용 및 시스템 아키텍처

시스템은 필수 조건에 의거하여 **라즈베리 파이 2대**를 구축하고, **Apache 웹 서버** 및 **Socket 통신**을 결합하여 하이브리드로 연동합니다.

```mermaid
graph TD
    User([사용자 브라우저]) -->|HTTP Access| Pi_A[Raspberry Pi A<br>Apache & PHP Web Server]
    Pi_A -->|1. REST API (cURL)| Pi_B[Raspberry Pi B<br>Python Flask Control Server]
    Pi_A -->|2. 긴급 제어 (Raw TCP Socket)| Pi_B
    
    subgraph Pi_B_Hardware [Pi B GPIO 하드웨어 제어]
        Pi_B -->|I2C or BCM| LCD[LCD1602 Display]
        Pi_B -->|GPIO| LED[RGB LED Indicator]
        Pi_B -->|PWM| Buzzer[Passive Buzzer]
    end
    
    Pi_B -->|CRUD| Storage[(JSON Store / SQLite DB)]
```

### A. Raspberry Pi A (Web Server)
* **역할**: 사용자 및 관리자를 위한 예약 현황 웹 인터페이스를 제공하고, Pi B의 백엔드 통신을 중계합니다.
* **주요 기술 및 연동 방식**:
  - **Apache2 & PHP**: 웹 호스팅 환경을 구성하고, PHP를 활용하여 백엔드 비동기 통신을 처리합니다.
  - **REST API 호출**: cURL 모듈을 사용해 Pi B의 Flask API 서버와 통신하여 강당 예약 목록의 CRUD(생성, 조회, 삭제) 연산을 신뢰성 있게 처리합니다.
  - **Raw TCP 소켓 송신 (필수 조건)**: 실시간 강제 상태 변경 등의 긴급 관리 명령이 떨어지면, PHP의 `fsockopen` 통신 모듈을 가동하여 Pi B의 소켓 포트(9000번)로 직접 Raw TCP 소켓 연결을 맺고 실시간으로 상태 전환 패킷을 발송합니다.

### B. Raspberry Pi B (Control Server)
* **역할**: 실제 강당 외부에 배치되어 임시 데이터베이스를 제어하고 하드웨어 GPIO 장치들을 드라이빙합니다.
* **주요 기술 및 연동 방식**:
  - **Python Flask REST API**: 포트 5000번을 대기하여 Pi A로부터 수신하는 예약 등록/삭제/상태 데이터 요청에 즉각 응답합니다.
  - **Python Multi-Thread Socket Server (필수 조건)**: 포트 9000번을 상시 오픈하여 Pi A로부터 유입되는 TCP 소켓 연결을 처리하고, 긴급 강제 상태 명령 수신 시 지연 시간(Latency) 없이 하드웨어에 즉시 반영합니다.
  - **실시간 스케줄러 (Scheduler Loop)**: 1초 단위로 시간 흐름을 판별하며 DB의 예약 시간 정보를 실시간 대조합니다. 회의 시작/종료 임계점 도달 시 LED, LCD, Buzzer 알림 장치를 작동시킵니다.
  - **저장소 추상화 (Repository Pattern)**: 개발 초기에는 단순한 파일 입출력 기반의 `JSON` 포맷 저장소를 사용하고, 안정적인 관리가 필요한 배포 단계에서는 설정 수정만으로 `SQLite DB`로 즉시 마이그레이션이 가능하도록 설계했습니다.

## 5. 사용 장비 및 기술 명세

| 구분 | 항목 | 수량 | 세부 규격 및 용도 |
| :--- | :--- | :--- | :--- |
| **하드웨어** | Raspberry Pi A | 1대 | Apache2, PHP 기반 사용자 예약 전용 웹 서버 |
| | Raspberry Pi B | 1대 | Flask API, Socket Server, GPIO 센서 제어용 하드웨어 제어기 |
| | LCD1602 디스플레이 | 1개 | I2C 백팩 모듈 탑재 및 Direct BCM 연결을 듀얼 지원하는 현황 패널 |
| | RGB LED (3색) | 1개 | Red/Green/Blue 단자를 매핑하여 강당 상태를 시각화 |
| | Passive Piezo Buzzer | 1개 | PWM 주파수를 인가해 도-미-솔 음계를 재생하는 알림 부저 |
| **소프트웨어**| 웹 서비스 | - | Apache 2.4, PHP 7.4+, Vanilla CSS (HSL & Glassmorphism 테마), JS |
| | 제어 백엔드 | - | Python 3.9+, Flask 2.0+, RPi.GPIO, RPLCD |
| | 데이터베이스 | - | JSON 파일 저장소, SQLite3 DB |
| **네트워크** | 하이브리드 통신 | - | REST API (HTTP Client-Server) & Raw TCP Socket (Port 9000) |

## 6. 기대 효과
1. **학생들의 사용 편의성 극대화**: 강당 앞을 지나거나 문을 열어보지 않아도, LED의 상태와 LCD 명세, 그리고 웹 화면을 통해 실시간 사용 상태를 직관적으로 확인할 수 있어 불필요한 동선과 대기 시간을 줄입니다.
2. **회의 및 행사 프라이버시 보호**: 행사 진행 중에 불쑥 문을 열고 확인하는 등의 돌발 상황을 예방하여, 회의 및 행사의 완성도와 프라이버시를 지켜줍니다.
3. **IoT 분산 통신 실습 기회**: 웹 서버(Apache/PHP)와 하드웨어 제어 서버(Flask)를 분리하고, 일반 비즈니스 로직(REST)과 비상 제어 로직(TCP Socket)을 분할하는 고도화된 하이브리드 IoT 분산 환경을 직접 구현하고 실증해 볼 수 있는 모범적인 사례를 제공합니다.
